# Feature Spec Template

## Goal
- この機能で達成したい目的を書く。

## Scope
- 対象に含めること
- 対象外にすること

## Backend
- 対応する module
- presentation / application / domain / infrastructure の観点で必要な実装

## Frontend
- 対応する page / feature / entity
- 必要な画面遷移、表示、送信処理

## Contracts
- 追加・更新する request / response / error contract
- `packages/contracts` を正とすること

## Rules
- 守るべき設計ルール
- 禁止事項

## Error handling
- 想定エラーと UI / API の扱い

## Acceptance criteria
- 完了判定に必要な条件
